# When solve TSP/VRP by OR-Tools, why distance_callback is only called once?

Today, I was using routing package from or-tools to solve a VRP problem. As showed in official document, the code are showed below.

Here is the template to solve a VRP, what we should do is just finish the distance_callback and assign some constraints.

But when I try to following documents to finish my program, I find that my distance_callback only runs once.

Reason: Since the routing solver does all computations with integers, the distance callback must return an integer distance for any two locations. If any of the entries of data['distance_matrix'] are not integers, you need to round either the matrix entries, or the return values of the callback, to integers. See Scaling the distance matrix for an example that shows how to avoid problems caused by rounding error.

Shortly, distance_callback function must return integer.

Which is written in the document. Unfortunately, I ignored it, which spends me many hours to figure out it.

# How to call distance_callback interior

I found that, if the model find the result of distance_callback is not it wanted (not integer), the model will stop, return a weird result but not error.

# It needs to calculate all elements of distance matrix

The tool for solving VRP needs to calculate all distance between two points, even it doesn't make a difference when solving problem, like the two farthest points.

Therefore, if you want to just calculate distance one at one time, it won't save your calculation.

Suggestion: calculate all distances before, and insert into sql table, then create index to accelerate searching.
