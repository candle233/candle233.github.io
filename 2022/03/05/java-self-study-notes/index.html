# Java 的三大版本：

  1. JAVA SE：是 JAVA 的标准版，基础和核心，主要来学习的一个部分。也是 JAVAEE 和 JAVAME 技术的基础，主要用于开发桌面应用程序。可以做简单桌面应用如：扫雷，连连看等。
  2. JAVA ME：JAVA 的微缩版，应用于嵌入式开发，比如手机程序开发。
  3. JAVA EE：是 JAVA 的企业版，提供企业级应用开发的完整解决方案，比如开发网站，企业的应用系统，是 JAVA 技术应用最广泛的领域。偏向于 WEB 的开发。

# JDK 和 JRE

JDK：
Java 2 SDK (Development Kit) 包含：JRE 的超集，包含编译器和调试器等用于程序开发的文件

JRE：
Java Runtime Environment (JRE) 包含：Java 虚拟机、库函数、运行 Java 应用程序和 Applet 所必须文件
Java 运行环境的三项主要功能：

  • 加载代码：由 class loader 完成；
  • 校验代码：由 bytecode verifier 完成；
  • 执行代码：由 runtime interpreter 完成。

区别和联系：

  • sdk（也就是 jdk）是 jre 的超集，是在 jre 的基础上增加了编译器及其他一些开发工具。
  • jre 就是 java 运行时环境， 包括了 jvm 和其它一些 java 核心 api 任何一台电脑， 只有安装了 jre 才可以运行 java 程序.

如果只是要运行 JAVA 程序， 之需要 JRE 就可以。JRE 通常非常小， 也包含了 JVM.
如果要开发 JAVA 程序， 就需要安装 JDK。

# 认识 JVM

（JAVA Virtual Machine）
JVM 是一种规范，可以使用软件来实现，也可使用硬件来实现，一个虚拟的用于执行 bytecodes 字节码的计算机。同时定义了指令集、寄存器集、结构栈、垃圾收集堆、内存区域。

JVM 负责将 java 字节码解释运行，边解释边运行，速度可能受到一定的影响。JAVA 提供了另一种解释运行的方法 JIT（just in time），可以一次解释完，再运行特定平台上的机器码，高级的 JIT 可以只分析热点代码，并将这些代码转成本地机器码，然后将结果缓存，下次直接从内存调用，提高了执行 JAVA 代码的效率，同时实现了跨平台、可移植的功能。

  1. JVM 是指在一台计算机上由软件或硬件模拟的计算机；它类似一个小巧而高效的 CPU。
  2. byte-code 代码是与平台无关的是虚拟机的机器指令。
  3. java 字节代码运行的两种方式:
    （1）方式 interpreter (解释)
    （2）Just-in-time (即时编译): 由代码生成器将字节代码转换成本机的机器代码，然后可以以较高速度执行。
    JAVA 的跨平台实现的核心是不同平台使用不同的虚拟机。不同操作系统虚拟机不同。Java 虚拟机机制屏蔽了底层运行平台的差别，从而实现 “一次编译，随处运行”。

# 位运算符

A = 0011 1100
B = 0000 1101
-----------------
A&b = 0000 1100
A | B = 0011 1101
A ^ B = 0011 0001
~A= 1100 0011
操作符描述
&如果相对应位都是 1，则结果为 1，否则为 0
如果相对应位都是 0，则结果为 0，否则为 1
^如果相对应位值相同，则结果为 0，否则为 1
~按位取反运算符翻转操作数的每一位，即 0 变成 1，1 变成 0。
<<按位左移运算符。左操作数按位左移右操作数指定的位数。
>>按位右移运算符。左操作数按位右移右操作数指定的位数。
>>>按位右移补零操作符。左操作数的值按右操作数指定的位数右移，移动得到的空位以零填充。

# 包机制

当定义了多个类的时候，可能会发生类名的重复问题。
在 java 中采用包机制处理定义类名的冲突问题。

  • 包的作用

    1、把相似或相关的类或接口组织在同个包中，方便查找和使用。
    2、像文件夹一样，包采用树形目录存储。一个包中的类名不同，不同的包中类名可以相同，同时调用不同包中相同类名的类时，需要加上包名来区别。
    3、包限定了访问权限，有包访问权限才能访问包中的类。

//包语句的语法格式为：
package pkg1[．pkg2[．pkg3…]];
//例如：
package net.java.util;
public class Something{   //  对应的路径保存在net/java/util/Something.java中
...
}

  • 关键字

    为了使用一个包中的成员，需要在使用前导入该包
    类文件中可以包含任意数量的 import 声明。import 声明必须在包声明之后，类声明之前。

//语法格式为：
import package1[.package2…].(classname|*);

  • ** 修饰符：** 修饰符，可选。告诉编译器如何调用该方法。定义了该方法的访问类型。
  • 返回值类型 ：方法可能会返回值。returnValueType 是方法返回值的数据类型。有些方法执行所需的操作，但没有返回值。在这种情况下，returnValueType 是关键字 void。
  • ** 方法名：** 是方法的实际名称。方法名和参数表共同构成方法签名。
  • ** 参数类型：** 参数像是一个占位符。当方法被调用时，传递值给参数。这个值被称为实参或变量。参数列表是指方法的参数类型、顺序和参数的个数。参数是可选的，方法可以不包含任何参数。
  • ** 方法体：** 方法体包含具体的语句，定义该方法的功能。

# 方法重载

如果你调用 max 方法时传递的是 int 型参数，则 int 型参数的 max 方法就会被调用；如果传递的是 double 型参数，则 double 类型的 max 方法体会被调用，这叫做方法重载；

# 选择结构

switch语句格式：
switch(表达式) {
    case 值1：
        语句体1;
        break;
    case 值2：
        语句体2;
        break;
        …//你可以有任意数量的case语句
    default：  //default语句表示所有情况都不匹配的时候，就执行该处的内容，和if语句的else相似。
        语句体n+1;
        break;    }

执行流程：
如果匹配成功的 case 没有 break ，从当前 case 开始，后续 case 的值都会输出，如果后续的 case 语句块有 break 语句则会跳出判断。
如果所有的 case 都和表达式的值不匹配，执行 default 语句，然后结束。

# Character 类

Character 类提供了一系列方法来操纵字符。你可以使用 Character 的构造方法创建一个 Character 类对象，例如：
Character ch = new Character('a');

方法描述
isLetter()是否为字母
isDigit()是否是一个数字字符
isWhitespace()是否是一个空白字符
isUpperCase()是否是大写字母
isLowerCase()是否是小写字母
toUpperCase()指定字母的大写形式
toLowerCase()指定字母的小写形式
toString()返回字符的字符串形式，字符串的长度仅为 1
